# Lessons — journal d'apprentissage

Chaque fichier markdown ici documente un apprentissage non-trivial :
- comportement particulier d'un site de résultats
- piège de matching sur un nom
- décision UI/UX importante
- erreur rencontrée et correctif

## Format

Nom de fichier : `YYYY-MM-DD-slug-court.md`

Contenu :
```
# Titre court

**Date :** YYYY-MM-DD
**Contexte :** (module / jalon concerné)
**Problème :** (ce qui a mal tourné ou ce qui a été compris)
**Leçon :** (règle à appliquer désormais)
**Preuve / lien :** (fichier:ligne, PR, ticket)
```
